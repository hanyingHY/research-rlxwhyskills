# Start Mode Routing

## Objective

Choose a justified starting posture instead of jumping directly into a full search program every time.

## Recommended outcomes

Use one of these starting outcomes:

1. `continue_full_workspace`
2. `continue_lite_workspace`
3. `bootstrap_then_full`
4. `bootstrap_then_lite`

## Continue full workspace

Choose this when the project already exposes a durable protocol surface such as:

1. phase state
2. user decision artifacts
3. worker runs or master tables
4. explicit route boards or validation tables

## Continue lite workspace

Choose this when the project already exposes a lightweight shortlist packet such as:

1. source index
2. retained evidence
3. action map
4. summary memo

## Bootstrap then lite

Prefer this when most of these are true:

1. the project is still a loose pile of notes, papers, or exports
2. conflict level appears low or still unknown
3. the main need is fast triage and decision support
4. the corpus is not obviously huge yet

## Bootstrap then full

Prefer this when one or more are true:

1. the corpus is already large
2. multiple versions, drafts, or conflicting findings are visible
3. the research clearly maps into downstream validation or execution
4. multiple workers or windows will likely be needed
5. auditability matters immediately

## Rule

If the project shows both heavy conflict and high downstream cost, do not hide behind a lite start just because it is smaller.

Choose the posture that protects the real execution risk.
