# Corpus Governance

## Objective

Keep the research corpus strict, auditable, and reversible.

## Default rule

Do not directly delete questionable material merely because it looks weak or incomplete.

Prefer this order:

1. downgrade
2. reject
3. quarantine
4. hard-delete only obvious junk after logging

## Why quarantine beats blind deletion

1. evidence programs need provenance
2. conflicts sometimes resolve in favor of previously weak-looking sources
3. duplicate or suspicious material still needs an audit trail
4. irreversible deletion based on vague quality standards weakens trust in the corpus

## Required fields for each rejection or quarantine action

1. item name
2. original path
3. primary direction
4. disposition status
5. reason
6. timestamp

## Approved hard-delete cases

Only hard-delete when all are true:

1. the file is empty, broken, or an obvious accidental duplicate
2. the action is logged first
3. no retained evidence depends on that copy
