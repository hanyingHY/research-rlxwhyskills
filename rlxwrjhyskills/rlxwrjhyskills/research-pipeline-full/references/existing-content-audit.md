# Existing Content Audit

## Objective

Do not trust pre-existing research content inside a project merely because it is already present on disk.

## Rule

If the workspace already contains research notes, reports, tables, or prior conclusions, run an authority and accuracy audit on that material before treating it as trusted evidence.

## Minimum audit questions

1. what claim is being made
2. what source or provenance supports it
3. is the source authoritative enough
4. does current external evidence confirm, weaken, or contradict it
5. should the existing content be retained, downgraded, quarantined, or rewritten

## Output requirement

Every substantial existing-content audit should leave behind:

1. a list of verified claims
2. a list of claims downgraded or contradicted
3. explicit reasons for each downgrade or correction
